## rewrite

In the past days i ran into something that i could not solve in [svelte](https://svelte.dev), so i moved on and gave [nextjs](https://nextjs.org/) a try. When i was building this websites, the biggest priority for me was to find a solution for the shit that `svelte` got me into. 

### nextjs over svelte?

#### svelte

Pros:
* easy to learn and use
* customizable and flexible
* Supports both client-side and server-side programming
* good performance 

Cons:

* lack of documentation
* limited support from the community
* hard to debug since all code is in one file
* requires a lot of boilerplate code for common tasks

#### nextjs

Pros:

* fast development time
* large, active community
* supports react
* good tooling

Cons:

* steep learning curve (can make it challenging for beginners)
* limited built in features
